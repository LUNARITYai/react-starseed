# GEMINI.md

This file provides context and instructions for Gemini CLI when working in the `react-starseed` repository.

## Project Overview

**React Starseed** is a modern, high-performance React 19 Single Page Application (SPA) built with Vite 6 and a focus on clean architecture and developer experience.

### Core Stack

- **Framework:** React 19 (Client-Side Rendering)
- **Build Tool:** Vite 6
- **Language:** TypeScript
- **Routing:** TanStack Router (File-based)
- **Data Fetching:** TanStack React Query v5 + Axios
- **Styling:** Tailwind CSS 4 (via `@tailwindcss/vite`) + shadcn/ui
- **Testing:** Playwright (E2E)
- **Package Manager:** pnpm (Node 22+ recommended)

## Architecture & Conventions

### Directory Structure

- `src/app.config.ts`: Global application metadata (name, tagline, URL).
- `src/routes/`: Contains file-based routes. `src/routeTree.gen.ts` is auto-generated by TanStack Router; do not modify it manually.
- `src/auth/`: Core authentication logic, including context provider, guards, and pluggable adapters (Mock, Better Auth, Supabase).
- `src/components/auth/`: Reusable authentication UI components (Login, Register, OAuth, etc.).
- `src/components/ui/`: Contains shadcn/ui primitives. Use the shadcn CLI to add or update these.
- `src/components/`: Contains custom, project-specific React components.
- `src/hooks/`: Custom React hooks (e.g., `useTheme.ts` for dark/light mode).
- `src/lib/`: Core utilities and library configurations:
  - `auth-config.ts`: Configuration for the authentication system and feature flags.
  - `utils.ts`: Exports `cn()` for conditional class merging (clsx + tailwind-merge).
  - `axios.ts`: Pre-configured Axios instance.
  - `query-client.ts`: TanStack QueryClient configuration.
- `src/styles/`: Global styles and Tailwind 4 design tokens (using OKLch variables).
- `tests/`: Playwright E2E tests.

### Key Conventions

- **Routing:** Follow file-based routing in `src/routes/`.
- **Metadata:** Use `src/app.config.ts` for global site metadata.
- **Authentication:** Pluggable auth system using `authAdapter` in `src/lib/auth-config.ts`. Protected routes use the `_authenticated` layout group in `src/routes/`.
- **Styling:** Use Tailwind CSS 4 utility classes. Prefer design tokens defined in `src/styles/global.css` over raw hex colors.
- **Theming:** Theme management is handled by `src/hooks/useTheme.ts`. State is persisted in `localStorage` and applied via the `.dark` class on the `<html>` element.
- **Components:** Always use the `cn()` utility for merging Tailwind classes.

## Development Workflow

### Commands

```bash
pnpm dev                  # Start the development server (http://localhost:5173)
pnpm build                # Production build (tsc + vite build)
pnpm preview              # Preview the production build (http://localhost:4173)
pnpm format               # Format code with Prettier
pnpm test:e2e             # Run Playwright E2E tests (headless)
pnpm test:e2e:ui          # Run Playwright E2E tests with UI
```

### Testing Guidelines

- E2E tests are located in the `tests/` directory.
- Playwright configuration is in `playwright.config.ts`.
- The preview server (`http://localhost:4173`) is the target for E2E tests.
- Ensure the project is built (`pnpm build`) before running tests in headless mode.

## Security & Safety

- **Secrets:** Never commit `.env` files or hardcode API keys. Use environment variables.
- **Dependencies:** Use **pnpm** exclusively to ensure consistent dependency resolution.
