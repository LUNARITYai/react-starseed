# React Starseed

An opinionated React starter template with file-based routing, data fetching, and a design-token-driven UI.

## Stack

| Category        | Technology                          |
| --------------- | ----------------------------------- |
| Framework       | React 19                            |
| Bundler         | Vite 6                              |
| Routing         | TanStack Router (file-based)        |
| Data fetching   | TanStack Query v5 + Axios           |
| Styling         | Tailwind CSS 4                      |
| UI primitives   | shadcn/ui (Radix UI)                |
| Icons           | Lucide React                        |
| Testing         | Playwright                          |
| Language        | TypeScript 5                        |
| Runtime         | Node ≥ 22                           |
| Package manager | pnpm                                |

## Quick start

```bash
# Clone
git clone https://github.com/LUNARITY/react-starseed.git
cd react-starseed

# Install dependencies
pnpm install

# Start the dev server (http://localhost:5173)
pnpm dev
```

## Available scripts

| Command                  | Description                         |
| ------------------------ | ----------------------------------- |
| `pnpm dev`               | Dev server at http://localhost:5173 |
| `pnpm build`             | Production build (tsc + Vite)       |
| `pnpm preview`           | Preview build at http://localhost:4173 |
| `pnpm format`            | Format with Prettier (write)        |
| `pnpm format:check`      | Check formatting                    |
| `pnpm test:e2e`          | Playwright E2E tests (headless)     |
| `pnpm test:e2e:ui`       | Playwright with UI                  |
| `pnpm test:e2e:debug`    | Playwright debug mode               |
| `pnpm test:e2e:install`  | Install Chromium for Playwright     |

Run a single test file:

```bash
pnpm exec playwright test tests/my-test.spec.ts
```


## Customization

### Site config

Edit `src/site.config.ts` to set your site name, tagline, and URL. Import from this file instead of hardcoding strings.

### Theming

Design tokens are defined as OKLch CSS variables in `src/styles/global.css` under `:root` and `.dark`. All shadcn/ui components consume these variables automatically. Extend the token system instead of adding raw hex colors.

Dark/light mode is managed by `src/hooks/useTheme.ts`, which stores the preference in `localStorage` and applies the `.dark` class on `<html>`. Flash prevention is handled by an inline script in `index.html`.

### Adding shadcn/ui components

```bash
pnpm dlx shadcn@latest add <component>
```

Do not edit files in `src/components/ui/` manually.

### Adding routes

Create a new file in `src/routes/` — the route tree is auto-generated by `@tanstack/router-plugin/vite`.

## Environment variables

Copy `.env.example` to `.env` and configure:

| Variable            | Default | Description              |
| ------------------- | ------- | ------------------------ |
| `VITE_API_BASE_URL` | `/api`  | Base URL for API requests |
